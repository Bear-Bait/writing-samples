* Executive Summary

March 6, 2026 at 08:02:16 AM, STATION-A 90.7-FM experienced 11 minutes of dead air. The outage was triggered by a stall in the "Democracy Now" external stream, which cascaded into a **Network Socket Exhaustion** event on the Automation iMac. A hidden hardware bottleneck—a consumer-grade ASIX USB Ethernet adapter—prevented the system from recovering. Investigation revealed the machine is not utilizing its native 1Gbps Broadcom port. The station recovered at 08:13:00 AM when Radiologik DJ was manually restarted.

** Key Metrics
- Detection time: ~1 minute
- Recovery time: 11 minutes
- Content delivered: Dead Air (Fill provided by manual switch to DN relay)
- Root cause confirmed: Yes (Kernel socket logs + throughput testing)

*** Technical Context
The `DJEvent-StreamErrorFor20Seconds.applescript` is currently an empty stub. When the stream stalled at 08:00:28, the system fired the error event, but since no recovery instructions existed, it sat on the stalled connection indefinitely.

** 08:02:16 - 08:13:00 AM: The Silent Failure (Socket Storm)

During this window, the application (PID 90057) remained alive but was deadlocked at the kernel level.

- **Kernel Logs:** Spammed with `ADDRINUSE: BSD wildcard with non-wild`.
- **Cause:** Radiologik attempted to reconnect thousands of times. Due to the buggy ASIX USB driver on `en8`, the OS failed to release old ports, leading to a total network stack lockup for the process.
- **I/O Interference:** `mds` (Spotlight) and `XProtectRemediator` were active, saturating Disk I/O and delaying the application's ability to resolve file paths for backup content.

** 08:13:00 AM: Recovery

- **Action:** Lead-Engineer manually killed the "ghost" Radiologik process and restarted the app.
- **Result:** Fresh PID 489 initialized and re-established the stream.

* Root Cause Analysis

** 1. Hidden Hardware Bottleneck (Primary Inhibitor)

The iMac is connected via an **ASIX AX88179A USB Adapter (`en8`)**. 
- **Performance:** Real-world throughput measured at **~69-100 Mbps**, despite negotiating at "Gigabit."
- **Comparison:** A Debian server on the same fiber line achieved **220+ Mbps**.
- **Driver Stability:** The ASIX chipset is notoriously unstable on Apple Silicon macOS 15+. It cannot handle high-frequency socket changes, leading to the `ADDRINUSE` kernel storm.

** 2. "Inactive" Native Interface (Design Oversight)

The iMac (Model `Mac16,3`) has a native Gigabit Ethernet port (`en0`) built into the **external power brick**. This port is currently **Inactive**. The station is using a $20 consumer adapter while the high-performance native Broadcom controller sits unused.

** 3. Cumulative System Fatigue (Maintenance Conflict)

The 8:00 AM window coincided with a "System Storm":
- **Spotlight Indexing:** Active mds/mds_stores threads were pinning the disk.
- **Zombie RSS feeds:** The scheduler is stuck in a 3-minute retry loop for feed `7yry17` ("File exists" error), creating constant background churn.

* Remediation

** Short-Term (Immediate)

*** Action 1: Reclaim Gigabit Speed (Hardware Fix)
*Priority: CRITICAL*
- **Task:** Physically unplug the Ethernet cable from the USB-C adapter and plug it into the **RJ45 port on the iMac power brick**.
- **Outcome:** Enables `en0` (Broadcom native), restores 1000Mbps throughput, and eliminates ASIX driver instability.

*** Action 2: Disable Contentious Indexing
*Priority: HIGH*
- **Action Taken:** Placed `.metadata_never_index` files in `Music/Radiologik` and `Library/Logs/Radiologik`.
- **Task:** Manually exclude the "BROADCAST-ORG Dropbox" folder via System Settings -> Spotlight -> Privacy.

** Medium-Term (System Stability)

*** Action 3: Populate Failsafe Scripts
*Priority: HIGH*
Update `DJEvent-StreamErrorFor20Seconds.applescript` to force a "Play Next" command to ensure the app doesn't sit on a stalled buffer.

*** Action 4: Clear "Zombie" RSS Feeds
*Priority: MEDIUM*
Audit the Scheduler for the `7yry17` and `2p8p05` feeds. These "File exists" errors are causing significant log bloat and scheduler thread contention.

* Postmortem Notes

The M4 iMac is an powerfull, but it is currently tripping over consumer-grade peripherals-USB-DONGLE to ETHERNET. The transition to the native Ethernet port (`en0`) is the single most important step to preventing another socket-exhaustion outage.

Use the correct ethernet port.
