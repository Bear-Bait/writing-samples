* Executive Summary

On February 23, 2026, Catskill B studio was inaccessible due to a **programmer logging into the wrong user account** (=wgxc_catskill_studio_b= instead of the current primary account). This caused Radiologik Scheduler to appear empty, Apple Music to show outdated playlists, and BUTT to fail connecting.

**IT resolution: 45 seconds.** The issue was identified via remote LogMeIn session and corrected immediately by logging into the correct account. All scripts, configurations, and playlists were intact on the primary account—nothing was lost or wiped.

The incident exposed a UX vulnerability: the presence of a legacy user account on the system can cause confusion if someone logs in to the wrong account. With correct account login, the system functioned normally.

* Timeline

** 2026-02-23 Evening: Discovery

Meredith Kooi (Catskill operations) found the studio non-functional:
- Radiologik Scheduler appeared empty (no scripts visible)
- Apple Music showing outdated playlists
- BUTT unable to connect to BROADCAST-ORG network
- Systems appeared "wiped"

** Investigation & Resolution (45 seconds)

Meredith logged in remotely via LogMeIn graphical interface. IT (Forrest) identified the issue immediately:

**Root cause:** A programmer had logged into the system using the legacy user account (=wgxc_catskill_studio_b=) instead of the current primary account.

**Resolution:** Log in to the correct primary user account.

**Time to fix:** 45 seconds. All scripts, playlists, and configurations were intact on the correct account. No data loss.

* Root Cause Analysis

A programmer remotely accessed Catskill B and inadvertently logged into the legacy user account (=wgxc_catskill_studio_b=) instead of the current primary account.

When the wrong account is active, the user sees only that account's data:
- Radiologik Scheduler appears empty (legacy account has no active scripts)
- Apple Music shows outdated playlists (legacy account's library)
- BUTT network configuration missing (only exists on primary account)

This created the false impression that the system was "wiped," when in reality all data was safe on the primary account.

**Resolution:** Log in to the correct primary user account. All systems immediately functional.

**Prevention consideration:** The presence of a legacy account on the system creates potential for this type of login confusion. Consolidating to a single account would eliminate this vulnerability.
